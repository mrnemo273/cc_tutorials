Step 1

Installing Claude

There are two things you'll want to install: the Claude Desktop App (the chat interface you'll use day-to-day) and Claude Code (the terminal-based coding agent that powers Figma-to-code workflows).

Install the Claude Desktop App

  1. Go to claude.com/download and download the installer for your operating system (Mac or Windows).
  2. Open the downloaded file and follow the on-screen instructions to install.
  3. Launch the app and sign in with your Anthropic account. If you don't have one, you can create a free account at claude.ai.
  4. That's it — you're ready to chat with Claude.

Install Claude Code

Claude Code is the terminal tool that connects to Figma and generates components. Don't worry — you won't need to be a terminal expert.

  1. Open your computer's Terminal app (Mac) or Command Prompt (Windows).
  2. Make sure you have Node.js installed. If not, download it from nodejs.org (pick the LTS version).
  3. Type the following command and press Enter:
npm install -g @anthropic-ai/claude-code
  1. Once installed, type claude in your terminal and press Enter. Follow the prompts to sign in with your Anthropic account.

Tip for designers: If the terminal feels intimidating, that's completely normal. You'll mostly just be pasting in a few commands. After the initial setup, Claude Code does the heavy lifting for you.

Step 3

Connecting Figma to Claude

The Figma MCP (Model Context Protocol) server is what lets Claude "see" your Figma designs — every color, spacing value, component variant, and layout detail. Here's how to set it up.

Option A: Remote Server (Easiest)

This is the recommended approach. No extra apps needed.

  1. Open your terminal and paste this command:
claude mcp add --transport http figma-remote-mcp https://mcp.figma.com/mcp
  1. Restart Claude Code if it was already running.
  2. In Claude Code, type /mcp and press Enter. You'll see the Figma server listed. Press Enter again to log in to your Figma account and click "Allow access."
  3. Type /mcp again to confirm the status shows "connected."

To make it available across all your projects, add the --scope user flag:

claude mcp add --scope user --transport http figma https://mcp.figma.com/mcp

Option B: Desktop Server (Uses the Figma App)

If you prefer using the Figma desktop app:

Open Figma Desktop, go to the Inspect panel, and click "Enable desktop MCP server." The server runs locally at http://127.0.0.1:3845/mcp.

Turning Figma Designs into Real Components.

Step 4

Getting Components from Figma into Claude Code

Once Figma MCP is connected, you can select any frame or component in Figma and ask Claude to turn it into code. Here's the basic workflow.

The Basic Flow

1. Copy the Figma Link

In Figma, right-click on any frame or component and choose "Copy link to selection." This gives Claude the exact node to look at.

2. Paste It into Claude Code

In your terminal running Claude Code, paste the link along with your request. For example: "Build a React component from this Figma frame: [paste link]"

3. Review and Iterate

Claude will generate the code. Review the output and ask for changes just like you would in a conversation — "make the padding larger" or "change the font to match our system."

Step 5

Building a Design System in Storybook

Storybook is a tool that lets you view and interact with your components in isolation — like a living catalog of your design system. Combined with Claude and Figma MCP, you can go from Figma components to a browsable Storybook library.

What is Storybook?

Think of Storybook as a visual playground for your components. Each component gets its own page where you can see all its states and variations, test it interactively, and document how to use it. It's the bridge between design and development.

Getting Started

Set up a new Storybook project

In your terminal, navigate to your project folder and run:

npx storybook@latest init

This sets up everything you need. Storybook will open in your browser automatically.

The Workflow: Figma to Storybook via Claude

1. Identify Your Components in Figma

Start with your most foundational components — buttons, inputs, cards, typography. Copy the Figma link for each one.

2. Ask Claude to Generate Component + Story

Paste the Figma link into Claude Code and ask it to create both a React component and a Storybook story file. For example: "Create a React component and Storybook story from this Figma button component: [link]"

3. Review in Storybook

Run npm run storybook and view your new component in the browser. Check every state and variant matches your Figma design.

4. Iterate with Claude

If something doesn't match, just tell Claude what needs to change. "The hover state should use our brand orange" or "Add a disabled variant."

Design Tokens + Prompts

Capturing Your Design System in Reusable Prompts.

Step 6

Design Tokens and Prompt Templates

Design tokens are the foundational values of your design system — colors, spacing, typography, radii, shadows. By capturing these in a prompt template, you give Claude the context it needs to generate components that match your system every time.

What Are Design Tokens?

Design tokens are named values that represent the small, repeated design decisions in your system. Instead of saying "use #EA633F," you say "use our brand-primary color." Claude can read these and apply them consistently across every component it generates.

Extracting Tokens from Figma

With Figma MCP connected, you can ask Claude to extract your tokens directly. Try pasting a link to your styles/variables page in Figma and asking:

Prompt: Extract Design Tokens

Look at this Figma file and extract all design tokens including:
- Color palette (primary, secondary, neutrals, semantic)
- Typography scale (font families, sizes, weights, line heights)
- Spacing scale (padding, margin, gap values)
- Border radius values
- Shadow definitions
- Breakpoints

Output them as a JSON tokens file and a CSS custom properties file.

Figma link: [paste your link here]

Reusable Prompt Templates

Save these prompts somewhere handy. Every time you need to generate a new component, start with the context prompt so Claude knows your system, then follow with the specific component request.

Prompt Template 1: Set Design System Context

You are helping me build components for our design system.
Here are our design tokens:

COLORS:
- Primary: [your hex]
- Secondary: [your hex]
- Background: [your hex]
- Surface: [your hex]
- Text Primary: [your hex]
- Text Secondary: [your hex]
- Border: [your hex]
- Error: [your hex]
- Success: [your hex]

TYPOGRAPHY:
- Font Family Headings: [your font]
- Font Family Body: [your font]
- Scale: h1=[size], h2=[size], h3=[size], body=[size], caption=[size]

SPACING:
- xs: [value]  sm: [value]  md: [value]  lg: [value]  xl: [value]

RADIUS:
- sm: [value]  md: [value]  lg: [value]  full: 9999px

Always use these tokens when generating components. Use CSS custom
properties (variables) that reference these values. Every component
should have a corresponding Storybook story showing all variants.

Prompt Template 2: Generate a Component from Figma

Using the design system context I gave you, create a React
component and Storybook story from this Figma design:

[paste Figma link]

Requirements:
- Use our design tokens (CSS custom properties)
- Include all variants visible in the Figma design
- Add prop types / TypeScript interfaces
- Create a Storybook story with controls for all props
- Include a "Docs" block describing usage guidelines
- Make it accessible (keyboard nav, ARIA labels, focus states)

Prompt Template 3: Audit Component Against Figma

Compare this existing component to the original Figma design
and list any differences:

Figma link: [paste link]
Component file: [paste file path or code]

Check for:
- Color accuracy (compare to our token values)
- Spacing and padding
- Typography (font, size, weight, line-height)
- Border radius and shadows
- Responsive behavior
- Missing states (hover, focus, disabled, loading)
- Accessibility issues

Prompt Template 4: Generate Full Token File from Figma

Analyze my Figma design system file and generate a complete
design tokens package:

Figma link: [paste link to your styles/variables page]

Output:
1. tokens.json — structured token file (compatible with Style Dictionary)
2. tokens.css — CSS custom properties
3. tokens.ts — TypeScript constants with full type safety
4. A summary table of all tokens organized by category

Group tokens by: color, typography, spacing, sizing, radius,
shadow, border, opacity, z-index, breakpoint.

Pro tip: Save your "Design System Context" prompt as a CLAUDE.md file in the root of your project. Claude Code automatically reads this file at the start of every session, so it will always know your design system without you having to paste the prompt each time.

You're Ready to Start.

The best way to learn is to pick one component from your Figma file and run through the full workflow. Start small, iterate, and build confidence one component at a time.
